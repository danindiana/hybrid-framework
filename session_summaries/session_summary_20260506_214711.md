# Session Summary - 2026-05-06 21:47:11

## Objective
Create a way to output what the Ollama loaded model is doing to the console and build a "wizard" to easily check it without disruption.

## Actions Taken
1.  **Investigation**: Confirmed Ollama is running as a systemd service (`ollama.service`).
2.  **Verification**: Verified that `journalctl -u ollama` provides access to the console logs and that `nvidia-smi` is available for GPU monitoring.
3.  **Implementation**: Created `ollama_wizard.sh`, a bash-based tool to:
    *   List currently loaded models using `ollama ps`.
    *   Monitor GPU usage across all devices.
    *   View recent logs or follow them in real-time using `journalctl`.
4.  **Testing**: Verified that `./ollama_wizard.sh status` and `./ollama_wizard.sh logs` work correctly.

## Tools Created
*   `ollama_wizard.sh`: The "Ollama Console Wizard" script.

## Usage
To use the wizard, run the following command from the workspace root:
```bash
./ollama_wizard.sh [status|logs|follow]
```
*   `status`: (Default) Shows loaded models and GPU stats.
*   `logs [n]`: Shows the last `n` lines of the Ollama log.
*   `follow`: Streams logs in real-time (ideal for watching model output/activity).
