## Topic: Troubleshooting Voice Input Software

### Research Phase
- Located 'voice-input' software in `/home/jeb/Documents/claude_creations/2026-04-02_voice-input`.
- Checked current configuration: `MIC_SOURCE="alsa_input.usb-UC03_UC03-00.mono-fallback"`.
- Checked available audio sources via `pactl` and `arecord`.
- **Finding**: The UC03 USB headset is physically connected (`lsusb`) and recognized by PipeWire (`wpctl`).
- **Test**: Successfully recorded 3 seconds of raw audio from the headset using `parec`.
- **Diagnosis**: The microphone is working correctly. However, the transcription script (`transcribe.py`) fails with `RuntimeError: CUDA failed with error out of memory`.
- **Root Cause**: Both GPUs (RTX 3080 and RTX 3060) are nearly full because of `ollama` processes, leaving insufficient VRAM for the "medium" Whisper model.

- **Final Diagnosis**: The microphone is fully functional. The software failure is a **VRAM exhaustion** issue.
- **Evidence**:
    - `wpctl` and `pactl` confirm the UC03 headset is active and unmuted.
    - Manual `parec` capture produced a file of the expected size.
    - `nvidia-smi` shows <1GB free on both GPUs due to a large Ollama model (`gemma4:31b`).
    - `transcribe.py` explicitly throws `CUDA out of memory` errors.

### Recommendations
- Unload the Ollama model: `ollama stop gemma4:31b-it-q4_K_M`.
- Or, modify `transcribe.py` to use `device="cpu"` (slower).
- Or, use a much smaller model (though even "base" failed with the current GPU load).
