# Camera Recording System Maintenance Summary - April 16, 2026

## 1. Issue: Camera Not Recording
The Hikvision camera was failing to trigger recordings despite motion events occurring.

### Root Cause Analysis
- **Stream Buffering:** The `curl` command used to monitor the camera's `alertStream` was buffering the incoming XML data. Because the events are small, the buffer didn't flush frequently enough for the script to react in real-time.
- **Process State:** Several orphaned `curl` and `ffmpeg` processes were found, likely from previous failed attempts or stream drops.

### Resolution
- **Unbuffered Stream:** Updated `record.sh` to use the `-N` (no-buffer) flag with `curl`.
- **Validation:** Confirmed that events are now processed immediately and `ffmpeg` triggers as expected. A test recording (`motion_2026-04-16_102341.mkv`) was successfully captured and verified.

## 2. Issue: Camera Time Desynchronization
The camera's internal clock was found to be off by approximately 10 days and 12 hours (reporting April 5th).

### Resolution
- **Manual Sync:** Performed an initial manual synchronization using the Hikvision ISAPI.
- **Automated Sync Logic:** Enhanced the `record.sh` script with a `sync_time` function that:
    - Synchronizes the camera clock at script startup.
    - Synchronizes every 12 hours during operation.
    - Synchronizes upon every alert stream reconnection.

The script now handles its own time maintenance, reducing the risk of timestamp mismatches in recorded files which could interfere with the 2-day retention cleanup logic.
